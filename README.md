## 📝 Notes Format

Every problem has structured notes in the following format:

### Problem Summary
A one or two line description of what the problem asks.

### Approach
A plain-English explanation of the strategy used — why this approach works and how it maps to the problem.

### Algo
Short numbered steps describing the algorithm. No extra explanation — just the steps.

### Key Insights
- Short bullet points highlighting the non-obvious observations that make the solution work.

### Dry Run
A step-by-step trace through a sample input to verify the logic visually.

### Observations
- Short bullet points on edge cases, tradeoffs, or alternative approaches worth noting.

### Time Complexity
Big-O analysis of the solution.

### Space Complexity
Big-O analysis of auxiliary space used.

### Points to Remember
- Short bullet points on reusable patterns and concepts.
- Things to recall when a similar problem appears in the future.
